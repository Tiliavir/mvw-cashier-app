# MVW Kassierer App

A lightweight, browser-based cashier app for club events ("Vereinsfeste"). Built with [Hugo](https://gohugo.io/) for templating and Vanilla JavaScript, stored entirely in `localStorage`, deployable on GitHub Pages.

---

## Features

- 📦 **No frameworks** — Vanilla JS (ES6) + Hugo static site generator
- 💾 **Offline-capable** — all data stored in `localStorage`
- 📱 **Mobile-first** responsive layout
- 🎟️ **Event management** — create and close events
- 🛒 **Item tiles** — color-coded, tap to add to cart
- 💶 **Transaction tracking** — total, received amount, change, tip
- 📊 **Settings & history** — view past events with revenue and transaction counts
- ⚖️ **Legal pages** — Impressum and Haftungsausschluss (liability disclaimer)
- 🗺️ **sitemap.xml + robots.txt** — auto-generated by Hugo

## Getting Started

### Prerequisites

- [Hugo](https://gohugo.io/installation/) (extended edition recommended)
- [Node.js](https://nodejs.org/) ≥ 20

### Install dependencies

```bash
npm install
```

### Build and serve locally

```bash
# Compile CSS + JS assets, then run Hugo
npm run compile

# Serve the compiled output
cd public && python3 -m http.server
```

Or use Hugo's built-in live-reload server (requires compiled assets in `static/` first):

```bash
npm run build:assets   # compile CSS + JS → static/
hugo server            # serves at http://localhost:1313/
```

## Build Pipeline

```
src/scss/style.scss  ──(sass + PostCSS)──► static/css/style.css  ─┐
src/js/**/*.js       ──(terser)──────────► static/**/*.js          ├─► Hugo ──► public/
content/ + layouts/  ──────────────────────────────────────────────┘
```

| Script | What it does |
|--------|-------------|
| `npm run build:css` | SCSS → `static/css/style.css` |
| `npm run build:js` | JS minification → `static/**/*.js` |
| `npm run build:assets` | CSS + JS (no Hugo) |
| `npm run build:hugo` | Hugo build → `public/` |
| `npm run compile` | All of the above |
| `npm run lint` | stylelint + eslint |
| `npm run validate` | vnu-jar HTML validation |
| `npm run test` | Unit tests |
| `npm run build` | compile + lint + validate + test |

## Deployment (GitHub Pages)

Push a tag to trigger a release and automatic deploy:

```bash
git tag v1.0.0
git push origin v1.0.0
```

The `release.yml` workflow:
1. Creates a GitHub Release with auto-generated release notes
2. Installs Hugo + Node.js dependencies
3. Compiles CSS/JS assets (`npm run build:assets`)
4. Runs `hugo --baseURL "$base_url/"` using the GitHub Pages base URL
5. Uploads and deploys `public/` to GitHub Pages

The Hugo `<base href>` approach handles subdirectory deployments (e.g. `https://username.github.io/mvw-cashier-app/`) correctly — no sed-based HTML patching needed.

## LocalStorage Reset

To clear all app data and start fresh, open the browser developer console and run:

```javascript
localStorage.removeItem('kassierer_app_v1');
location.reload();
```

---

## Known Limitations

- **No multi-device sync** — data lives only in the browser's `localStorage`
- **No server backup** — if browser data is cleared, all data is lost
- **Single browser** — each device/browser has its own isolated data
